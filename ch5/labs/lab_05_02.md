# Lab 05-2

## 情境

電商網站的購物車程式需要根據商品 ID 清單建立陣列、加入商品、刪除商品，最後印出目前購物車內容。

不過，這段程式有兩個問題：

1. 使用 `Array()` 的方式可能產生非預期結果
2. 使用 `delete` 刪除陣列元素後，會留下空洞 (empty item)

請你閱讀並修改下面的程式碼，讓它符合需求。

## Starter Code

```js
// 商品 ID 來自外部系統
let itemIds = [5];

// 1. 建立購物車
let cart = new Array(...itemIds);

// 2. 再加入兩個商品
cart.push(12);
cart.push(18);

// 3. 刪除第 2 個商品
delete cart[1];

// 4. 印出購物車內容
for (let i = 0; i < cart.length; i++) {
  console.log(`Item ${i + 1}: ${cart[i]}`);
}
```

## 任務

請修改程式碼，完成以下要求：

1. `cart` 一開始應該是只有一個元素的陣列 `[5]`
2. 刪除第 2 個商品後，陣列中不能留下 empty item
3. 最後輸出結果必須是：

```js
Item 1: 5
Item 2: 18
```

4. 使用以下其中一種方式建立 `cart`
   - `Array.of(...itemIds)`
   - `[...itemIds]`

5. 使用以下其中一種方式輸出結果
   - `for...of` 搭配 `entries()`
   - `forEach()`

之後，傳修改後的程式碼至 Tronclass. 

## 問題

完成程式後，回答以下問題：

1. 為什麼 `new Array(...itemIds)` 在 `itemIds = [5]` 時會產生問題？
2. 為什麼 `splice()` 比 `delete` 更適合用來刪除陣列元素？

## 提示

- `Array()` 在只有一個數字引數時，代表的是「長度」
- `delete` 不會改變陣列長度
- `splice(start, deleteCount)` 會真正移除元素
- `entries()` 可以同時取得 index 和 value
