# Lab 06-04：使用 Factory Pattern 建立限次優惠券

## 實驗目標

1. 理解工廠函數會回傳新的函數物件。
2. 使用閉包保存函數內部狀態。
3. 建立多個狀態彼此獨立的函數物件。

## 任務說明

電子商務網站會發放不同的優惠券。每張優惠券都有自己的：

- 優惠券代碼
- 折扣金額
- 可使用次數

請完成一個工廠函數 `createCoupon(couponCode, discountAmount, maxUses)`：

- 每次呼叫 `createCoupon` 都會建立一個新的優惠券函數。
- 優惠券函數會接收一個訂單金額 `orderAmount`。
- 如果優惠券還有可用次數，就扣掉折扣金額，並減少一次可用次數。
- 如果優惠券已經沒有可用次數，就回傳原本訂單金額。
- 不同優惠券的可用次數必須彼此獨立。

## 預期輸出

```text
NEW100 折扣後金額：900，剩餘 1 次
NEW100 折扣後金額：700，剩餘 0 次
NEW100 已無可用次數，金額：500
VIP200 折扣後金額：1000，剩餘 0 次
VIP200 已無可用次數，金額：1200
```

## 干擾片段

以下片段不需要使用：

```javascript

```

```javascript

```

## 提示

- `createCoupon` 是工廠函數。
- `remainingUses` 應該宣告在 `createCoupon` 裡面，讓每張優惠券有自己的可用次數。
- 被 `return` 回傳的內部函數會形成閉包，因此仍然可以存取 `couponCode`、`discountAmount` 和 `remainingUses`。

## 概念問題

1. 為什麼 `createCoupon` 是一個 factory function？
2. `newUserCoupon` 和 `vipCoupon` 的 `remainingUses` 會不會互相影響？為什麼？
3. 為什麼外部程式碼不能直接修改 `remainingUses`？
