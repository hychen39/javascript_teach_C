# 實驗 07-01：商品狀態顯示

## 目標

請閱讀一段電商商品狀態顯示程式，找出物件屬性存取錯誤，並修改成可讀性較好的寫法。

本實驗練習：

- 讀懂 class、constructor、實例屬性與實例方法。
- 判斷何時使用點運算子 `.`，何時使用方括號 `[]`。
- 修正動態屬性存取的錯誤。
- 避免在固定屬性上使用 magic string。

## 情境

電商後台需要根據商品的庫存狀態，顯示對應的文字與顏色。

目前程式碼可以建立商品物件，但商品狀態顯示有問題。請閱讀程式碼，分析錯誤原因，並完成修改。

## Starter Code

```js
const STATUS_CONFIG = {
  available: { text: '可購買', color: 'green' },
  low_stock: { text: '庫存不足', color: 'yellow' },
  sold_out: { text: '售完', color: 'red' }
};

class Product {
  constructor(name, price, stockStatus) {
    this.name = name;
    this.price = price;
    this.stockStatus = stockStatus;
  }

  showStatus() {
    const statusInfo = STATUS_CONFIG.stockStatus;

    console.log(this['name']);
    console.log(statusInfo['text']);
    console.log(statusInfo['color']);
  }
}

const product = new Product('無線滑鼠', 799, 'low_stock');
product.showStatus();
```

## 任務

### R1. 讀程式碼並分析問題

請回答：

1. `product` 物件建立後，`product.stockStatus` 的值是什麼？
2. `STATUS_CONFIG.stockStatus` 實際上是在找哪一個屬性？
3. 為什麼這個寫法無法取得 `low_stock` 對應的狀態設定？

### R2. 修正動態屬性存取

修改 `showStatus()`，讓程式可以根據 `this.stockStatus` 動態取得 `STATUS_CONFIG` 中對應的設定。

修正後的輸出應為：

```txt
無線滑鼠
庫存不足
yellow
```

### R3. 改善固定屬性的存取寫法

在 `showStatus()` 中，將固定屬性的存取方式改成點運算子 `.`。

請思考：

- `this['name']` 應該改成什麼？
- `statusInfo['text']` 應該改成什麼？
- `statusInfo['color']` 應該改成什麼？

### R4. 加入未知狀態的預設值

如果 `stockStatus` 是未知值，請讓程式使用以下預設狀態：

```js
{ text: '未知狀態', color: 'gray' }
```

例如：

```js
const product = new Product('藍牙鍵盤', 1290, 'preorder');
product.showStatus();
```

預期輸出：

```txt
藍牙鍵盤
未知狀態
gray
```
